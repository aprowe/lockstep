A — Single-page, hero-forward Recommended

One long scrolling page. Big hero with app screenshot, feature highlights, 2-column pricing (Free vs $30), donate + GitHub in footer.

  • No routing complexity, pure Next.js static export
  • Standard pattern for indie dev tools (Raycast, Warp, etc.)
  • Easy to deploy to Vercel as-is
  • Everything on one URL — can't deep-link to pricing

B — Minimal one-pager

Stripped down. Hero + tagline + 3 features + inline pricing. Looks intentionally terse — like the README turned website.

  • Fast to build, easy to maintain
  • Distinctive — most tools over-explain
  • Less room to showcase screenshots and workflow
  • Harder to sell the $30 commercial license without context

C — Multi-page (Home / Pricing / Docs)

Separate pages for home, pricing, and optional docs. More room for each section; works better if content grows.

  • Deep-linkable pricing and docs URLs
  • Scales as the product matures
  • More routing to wire up; slower to build
  • Overkill for a pre-release indie tool
